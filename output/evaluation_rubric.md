# PDF→Markdown 변환 품질 평가 루브릭

## 평가 대상
- 원본: TV-RAG 학술 논문 PDF
- 변환 도구: markitdown, pdftotext, pymupdf, mineru

---

## 평가 기준 (총 100점)

### 1. 텍스트 정확성 (20점)
원문의 텍스트가 얼마나 정확하게 추출되었는가

| 점수 | 기준 |
|------|------|
| 20 | 텍스트 오류 없음, 완벽한 추출 |
| 15 | 경미한 오류 (1-5개), 의미 전달에 문제 없음 |
| 10 | 중간 정도 오류 (6-15개), 일부 문장 이해 어려움 |
| 5 | 심각한 오류, 다수의 문장 깨짐 |
| 0 | 텍스트 대부분 손실 또는 심각하게 깨짐 |

### 2. 구조 보존 (20점)
제목, 섹션, 서브섹션 등의 문서 구조가 잘 보존되었는가

| 점수 | 기준 |
|------|------|
| 20 | 모든 제목/섹션이 올바른 마크다운 문법으로 표현 (`#`, `##` 등) |
| 15 | 대부분 구조 보존, 일부 레벨 오류 |
| 10 | 구조는 인식되나 마크다운 문법 미적용 |
| 5 | 구조 일부만 보존, 대부분 plain text |
| 0 | 구조 완전 손실 |

### 3. 수식 처리 (15점)
수학 공식이 올바르게 표현되었는가

| 점수 | 기준 |
|------|------|
| 15 | LaTeX 형식으로 완벽하게 변환 (`$...$`, `$$...$$`) |
| 12 | LaTeX 변환되었으나 일부 오류 |
| 8 | 수식 구조는 보이나 렌더링 불가 |
| 4 | 수식 대부분 깨짐, 특수문자 아티팩트 |
| 0 | 수식 완전 손실 또는 의미 파악 불가 |

### 4. 표 처리 (15점)
표가 올바른 형식으로 변환되었는가

| 점수 | 기준 |
|------|------|
| 15 | 마크다운 테이블 또는 HTML 테이블로 완벽 변환, 데이터 정확 |
| 12 | 테이블 형식 유지, 일부 셀 병합/정렬 오류 |
| 8 | 테이블 구조 인식되나 형식 깨짐 |
| 4 | 테이블 데이터만 추출, 구조 손실 |
| 0 | 테이블 완전 손실 |

### 5. 이미지 참조 (10점)
이미지/그림 참조가 어떻게 처리되었는가

| 점수 | 기준 |
|------|------|
| 10 | 이미지 경로가 마크다운 문법으로 참조 (`![](path)`) |
| 7 | 이미지 참조 있으나 경로 오류 또는 일부 누락 |
| 4 | 이미지 위치만 표시 (placeholder) |
| 0 | 이미지 참조 완전 누락 |

### 6. 가독성 (10점)
마크다운 문서의 전반적인 가독성

| 점수 | 기준 |
|------|------|
| 10 | 바로 렌더링 가능, 읽기 편함 |
| 7 | 약간의 정리 필요하나 대체로 읽기 좋음 |
| 4 | 상당한 정리 필요, 2단 레이아웃 혼란 등 |
| 0 | 읽기 매우 어려움 |

### 7. 노이즈/아티팩트 (10점)
변환 과정에서 발생한 불필요한 문자나 아티팩트

| 점수 | 기준 |
|------|------|
| 10 | 노이즈 없음 |
| 7 | 경미한 노이즈 (특수문자 1-5개) |
| 4 | 중간 정도 노이즈, 일부 섹션에 아티팩트 |
| 0 | 심각한 노이즈, 메타데이터 오염, 의미없는 문자 다수 |
